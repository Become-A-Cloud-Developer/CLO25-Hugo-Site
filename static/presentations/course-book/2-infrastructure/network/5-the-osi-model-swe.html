Vad är OSI-modellen?

Ett konceptuellt ramverk som organiserar nätverkskommunikation i sju distinkta lager

Varje lager hanterar en specifik aspekt: fysisk överföring, routing, pålitlig leverans, applikationsprotokoll

Lagren tillhandahåller tjänster till lagret ovanför och konsumerar tjänster från lagret nedanför

Separationen tillåter protokoll att ändras utan att påverka andra lager

Lager 1-2: Physical & Data Link

Lager 1 (Physical): Rå bitöverföring—elektriska signaler, kablar, fiberoptik, trådlösa radios

Ingen förståelse för adresser eller paket; överför/tar endast emot bitströmmar

Lager 2 (Data Link): Organiserar bitar i ramar; använder MAC addresses för lokal leverans

Switches arbetar på lager 2, vidarebefordrar ramar baserat på MAC addresses

Lager 3: Network Layer

Möjliggör routing över flera nätverk med hjälp av IP addresses

Routers arbetar på lager 3, konsulterar routing-tabeller för att vidarebefordra paket

Hanterar paket fragmentering när data överstiger länkkapaciteten

ICMP tillhandahåller diagnostik (ping, traceroute)

Lager 4: Transport Layer

Hanterar end-to-end kommunikation; använder portnummer för att identifiera applikationer

TCP: Anslutningsorienterad, garanterad leverans, sekvenshantering, återöverföring

UDP: Anslutningslös, ingen leveransgaranti, lägre latens, mindre overhead

Välj TCP när dataintegritet är viktig; UDP när hastighet är viktigare än enstaka förlust

Lager 5-7: Session, Presentation, Application

Lager 5 (Session): Hanterar anslutningsetablering, underhåll och avslut

Lager 6 (Presentation): Dataformatöversättning, kryptering/dekryptering, komprimering

Lager 7 (Application): Nätverkstjänster för användarapplikationer—HTTP, HTTPS, DNS, SSH, SMTP

Många applikationer integrerar session/presentation direkt i applikationsprotokoll

Load Balancers: Layer 4 vs Layer 7

Layer 4 Load Balancer: Routar baserat på IP addresses och portar; snabb, minimal inspektion

Layer 7 Load Balancer: Parsar HTTP—routar efter URL-sökväg, headers, cookies; möjliggör sofistikerade regler

L7 kan routa /api/* till API-servrar, /static/* till CDN; terminera SSL; inspektera för hot

Många distributioner använder båda: L4 distribuerar till L7 balancers, som routar till backends

Viktig insikt

OSI-modellen tillhandahåller ett systematiskt ramverk för att förstå nätverkskommunikation—från fysiska signaler till applikationsprotokoll—vilket möjliggör välgrundade beslut om arkitektur och felsökning
