BRANDVÄGGAR

Infrastrukturgrunder

Vad är en brandvägg?

En firewall fungerar som en barriär mellan betrodda interna nätverk och obetrodda externa

Granskar varje paket som korsar nätverksgränsen och beslutar att tillåta eller blockera

Beslut följer regler som definieras av administratörer baserat på paketattribut

Om ingen regel matchar avgör en standardpolicy resultatet (vanligtvis "neka alla")

Five-tupeln

Brandväggsregler matchar vanligtvis fem paketattribut (five-tuple):

Käll-IP-adress: Det ursprungliga systemets nätverksadress

Destinations-IP-adress: Målsystemets nätverksadress

Käll-/destinationsportar: Identifierar tjänsten (t.ex. 443 för HTTPS)

Protokoll: Transportprotokoll—vanligtvis TCP eller UDP

Portar och tjänster

Nätverkstjänster lyssnar på specifika portar—numeriska identifierare som dirigerar trafik till applikationer

Port 22: SSH (säker fjärråtkomst)

Port 80/443: HTTP/HTTPS (webbtrafik)

Port 3306: MySQL; Port 5432: PostgreSQL

Portbaserad filtrering beviljar eller nekar åtkomst till hela tjänster

Brandväggsarkitekturer

Paketfiltrering: Granskar paket individuellt; snabbt men ingen anslutningsmedvetenhet

Stateful Inspection: Spårar anslutningstillstånd; validerar att trafik tillhör etablerade sessioner

Nästa generations (NGFW): Applikationslagermedvetenhet, djup paketinspektion, hotdetektering

De flesta moderna brandväggar arbetar statefully som standard för meningsfull säkerhet

Brandväggsregler

Regler specificerar: Riktning (inkommande/utgående), Källa/destination, Port/protokoll, Åtgärd

Regler bearbetas i ordning—första matchningen vinner, sedan stoppar bearbetningen

Vanligt mönster: specifika tillåt-regler först, sedan bredare neka-regler, standardnekande sist

Exempel: Tillåt SSH från hanterings-IP → Neka SSH från alla → Tillåt HTTP/HTTPS → Standardnekande

Azure Network Security Groups

NSG tillhandahåller stateful paketfiltrering för virtuella datorer och subnät i Azure

Regler inkluderar prioritetsnummer—lägre värden körs först

Service Tags representerar Azure-tjänsters IP-intervall utan att ange exakta adresser

Application Security Groups möjliggör regler baserade på arbetsbelastningsfunktion, inte bara IP-adresser

Viktig insikt

Brandväggar kontrollerar trafik genom att utvärdera paket mot definierade regler—effektiv konfiguration kräver specifika regler, lämpliga standardinställningar och djupgående försvar över nätverkslager
