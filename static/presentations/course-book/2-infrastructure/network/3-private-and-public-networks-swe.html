Infrastrukturgrunder

Adressutrymmesproblemet

IPv4 tillhandahåller cirka 4,3 miljarder unika adresser—otillräckligt för modern enhetsproliferation

Istället för att kräva globalt unika adresser för varje enhet, definierar standarder privata intervall

Privata adresser kommunicerar inom lokala nätverk men kan inte routa över det publika internet

Detta bevarar publika adresser för system som behöver global nåbarhet

Privata nätverk

Använder icke-routbara IP-adresser för intern kommunikation (RFC 1918)

10.0.0.0/8: ~16 miljoner adresser för stora organisationer

172.16.0.0/12: ~1 miljon adresser för medelstora nätverk

192.168.0.0/16: 65 536 adresser vanliga i hem/små kontor

Dessa intervall kan återanvändas över organisationer utan konflikt

Hur privata nätverk fungerar

Enheter får privata IP-adresser (t.ex. 192.168.1.10, 192.168.1.20) och kommunicerar direkt

NAT (Network Address Translation) skriver om källadresser när trafik lämnar nätverket

Flera enheter delar en enda publik IP—externa system ser endast routerns adress

Routern upprätthåller en översättningstabell som mappar interna enheter till anslutningar

Publika nätverk

Använder globalt unika, routbara IP-adresser tilldelade av ISP:er/molnleverantörer

Statiska publika IP-adresser: Förblir konstanta; idealiska för servrar med DNS-poster

Dynamiska publika IP-adresser: Kan ändras vid återanslutning; vanliga för hemuppkopplingar

Vilket system som helst med internetanslutning kan potentiellt nå publika adresser

Anslutbarhet och exponering

Publika IP-adresser är globalt nåbara—möjliggör tjänster men kräver robust säkerhet

Firewalls filtrerar inkommande anslutningar, tillåter endast specifika portar/protokoll

Network Security Groups definierar regler innan trafik når resurser

DMZ (Demilitarized Zone): Publikt exponerade system isolerade från det interna nätverket

Att välja mellan privat och publikt

Publik IP: Webbservrar, jump boxes, API-gatewayer—måste acceptera externa anslutningar

Privat IP: Databaser, applikationsservrar, interna filservrar—ingen direkt internetåtkomst behövs

Vanligt mönster: Load balancer med publik IP distribuerar till privata backend-servrar

Exponera endast det som måste vara publikt; allt annat opererar på privata nätverk

Viktig insikt

Privata nätverk ger isolering och adressbesparing medan publika nätverk möjliggör global anslutbarhet—de flesta arkitekturer kombinerar båda med NAT och firewalls vid gränsen
