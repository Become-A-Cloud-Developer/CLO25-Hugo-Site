Infrastrukturgrunder

Server-konceptet

En server tillhandahåller compute-resurser som bearbetar förfrågningar och levererar tjänster över ett nätverk

Hårdvarudefinition: En fysisk dator optimerad för kontinuerlig drift och nätverkstillgänglighet

Mjukvarudefinition: Ett program som lyssnar efter förfrågningar och skickar svar (web server, database server, application server)

Båda definitionerna samexisterar: "database server" syftar både på PostgreSQL-processen och maskinen som kör den

Client-Server Model

Ett system (server) tillhandahåller tjänster som andra system (clients) konsumerar

Centraliserad resurshantering: En enskild server underhåller den auktoritativa kopian av data och tjänster

Skalbarhet: Att lägga till clients kräver inte duplicering av kärntjänsten—bara serverns kapacitet att hantera samtidiga förfrågningar

Exempel: Webbläsare begär sidor från en web server, mobilapp frågar en API server, backend hämtar data från en database server

Server Deployment-modeller

Samma applikationslogik kan köras på olika infrastrukturgrunder

Physical Servers: Dedikerad hårdvara, maximal prestanda, fullständig kontroll

Virtual Machines: Mjukvarubaserad emulering, bättre utnyttjande, snabb provisionering

Containers: Lättviktig isolering, millisekund startup, effektiv skalning

Serverless Functions: Händelsedriven exekvering, automatisk skalning, ingen infrastrukturhantering

Physical Servers & Virtual Machines

Physical Servers (Bare-Metal): OS körs direkt på hårdvara, maximal prestanda, operationell overhead (upphandling, installation, underhåll)

Virtual Machines: Hypervisor-lager hanterar hårdvara, flera VMs på samma host, varje VM har sin egen OS-instans

VM-fördelar: Bättre hårdvaruutnyttjande, snabb provisionering (minuter vs veckor), förbrukningsbaserad prissättning i cloud-plattformar

VM-avvägningar: 2-10% prestandaoverhead, full OS-vikt per instans, OS-underhåll för varje VM

Containers & Serverless

Containers: Paketerar applikation med beroenden, delar host OS kernel, isolerade processer

Container-fördelar: Lättviktig (millisekund startup), konsekventa miljöer över dev/test/prod, effektiv skalning med orkestrering (Kubernetes)

Serverless Functions: Kod exekveras som svar på händelser (HTTP request, queue message, timer), plattformen hanterar all infrastruktur

Serverless-fördelar: Inga servrar att hantera, automatisk skalning, pay-per-execution-fakturering, idealisk för varierande arbetsbelastningar

Serverless-begränsningar: Exekveringstidsbegränsningar (5-15 min), cold start-latens, bäst för stateless-operationer

Server-roller

Web Servers: Accepterar HTTP/HTTPS-förfrågningar, serverar statiska filer eller proxar till application servers (nginx, Apache)

Application Servers: Exekverar affärslogik, genererar dynamiskt innehåll, bearbetar användarförfrågningar (Gunicorn, uWSGI för Python)

Database Servers: Hanterar persistent datalagring, upprätthåller integritet, optimerar queries, hanterar samtidig åtkomst (PostgreSQL, MySQL)

File Servers: Tillhandahåller centraliserad lagring över nätverk, implementerar åtkomstkontroller (NFS, SMB, eller cloud object storage)

Produktionssystem kombinerar typiskt flera roller som arbetar tillsammans—förfrågningar flödar från web server → application server → database server
