Vad är Server-roller?

Servrar kategoriseras efter de tjänster de tillhandahåller. Istället för monolitiska system distribuerar organisationer servrar med definierade roller optimerade för specifika uppgifter.

Denna arkitektur möjliggör oberoende skalning, felisolering och optimerad prestanda för varje komponent.

Översikt över Server-roller

Database Server - Lagrar och hanterar strukturerad data med integritetskrav

File Server - Centraliserad fillagring med nätverksåtkomst och behörigheter

Mail Server - Hanterar e-postoperationer: skicka, ta emot, dirigera, lagra

DNS Server - Översätter domännamn till IP-adresser

DHCP Server - Tilldelar dynamiskt IP-adresser till nätverksenheter

Web Servers

Accepterar HTTP/HTTPS requests från klienter och returnerar svar

Statiskt innehåll (HTML, CSS, bilder) - serveras direkt från disk

Dynamiskt innehåll - delegeras till application servers för bearbetning

Huvudfunktioner: SSL/TLS termination, svarskomprimering, caching, request routing, hantering av tusentals samtidiga anslutningar

Exempel: nginx (event-driven), Apache (process-baserad)

Application Servers

Utför affärslogik och genererar dynamiskt innehåll

Tillhandahåller runtime environment för applikationskod: Python (Gunicorn, uWSGI), Java (Tomcat, WildFly)

Hanterar request-bearbetning, databasanslutningar, sessionshantering, resursallokering

Sitter vanligtvis bakom web servers - web server hanterar SSL, statiskt innehåll, routing; application server exekverar logik

Skalar horisontellt - distribuera flera instanser bakom load balancer

Reverse Proxies

Sitter mellan klienter och backend-servrar, agerar på servrars vägnar

Load balancing - distribuerar requests över flera backends (round-robin, least connections, weighted)

SSL termination - avlastar kryptering/dekryptering från backends

Caching - lagrar svar och serverar direkt för identiska requests

Request routing - dirigerar till olika backends baserat på URL, headers, kriterier

Exempel: nginx, HAProxy, Azure Application Gateway, AWS ALB

Bastion Hosts

Tillhandahåller säker administrativ åtkomst till servrar i privata nätverk

Kontrollerad ingångspunkt - enda servern med både extern tillgänglighet och intern nätverksåtkomst

Reducerad attackyta - interna servrar har inga administrativa portar exponerade mot internet

Centraliserad åtkomstkontroll - en enda punkt för autentisering, MFA-tillämpning

Audit logging - alla administrativa sessioner loggas på en enda plats

Molntjänster: Azure Bastion, AWS Systems Manager Session Manager

Hur dessa roller samverkar

Typiskt flöde för en webbapplikation:

1. Klienter ansluter till reverse proxy (SSL termination, routing)

2. Statiskt innehåll serveras direkt; dynamiska requests vidarebefordras till application servers

3. Application servers utför affärslogik, frågar database servers

4. Administratörer får åtkomst till alla servrar genom bastion host

Arkitekturen skalar från en enda VM (alla roller kombinerade) till distribuerade system (oberoende skalning per roll)
