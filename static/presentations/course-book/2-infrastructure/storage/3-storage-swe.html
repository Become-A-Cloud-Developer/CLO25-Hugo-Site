LAGRING

Infrastruktur Grunder

Vad är Lagring?

Lagring avser system och enheter som används för att spara, hämta och hantera data

Kritiskt för applikationer, system och datadrivna operationer

Skiljer sig från databaser: lagring tillhandahåller infrastruktur, databaser lägger till struktur och sökbarhet

Grund för persistenslagret i alla systemarkitekturer

Kapacitet och Prestanda

Kapacitet: Den totala mängden data ett lagringssystem kan hantera

Mäts i GB, TB eller PB för större system

Throughput: Mängden data som överförs per sekund (MB/s, GB/s)

Latens: Tid det tar att få åtkomst till data efter en begäran (ms, μs)

Redundans

Redundans replikerar data för att kunna återställa vid fel

RAID: Kombinerar flera diskar för att förbättra prestanda, kapacitet och redundans

Snapshots: Ögonblicksbilder av data som lagras lokalt eller nära källan

Backuper: Regelbundna kopior av data som lagras externt för återställning

Block Storage

Delar upp data i block av fast storlek

Varje block har en unik adress, åtkomst sker oberoende

Används för databaser och applikationer som kräver direkt diskåtkomst

Protokoll: iSCSI, Fibre Channel, NVMe

Exempel: Hårddiskar, SAN (Storage Area Network), Azure Disk Storage

File Storage

Hanterar filer och kataloger likt ett hierarkiskt filsystem

Perfekt för delad åtkomst över flera system

NAS (Network-Attached Storage) tillhandahåller centraliserade filtjänster

Protokoll: SMB/CIFS (Windows), NFS (Unix/Linux)

Exempel: Azure Files, Amazon EFS, traditionella NAS-enheter

Object Storage

Lagrar data som objekt med metadata, likt filer men utan hierarki

Platt namnrymd med buckets/containers istället för mappar

Lämplig för ostrukturerad data i massiv skala

Åtkomst via HTTP/S API:er (REST)

Exempel: Azure Blob Storage, Amazon S3, Google Cloud Storage

Praktiska Tillämpningar

Datasjöar: Big data-analys och maskininlärning

Medialagring: Videoströmningsplattformar, bildarkiv

Backup och arkivering: Långtidslagring och efterlevnad

Applikationsdata: VM-diskar, containervolymer, databaslagring

Sammanfattning

Lagring är en grundläggande komponent i IT-infrastruktur. Att välja rätt lagringstyp förbättrar prestanda, skalbarhet och tillförlitlighet.
