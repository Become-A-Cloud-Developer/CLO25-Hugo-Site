Infrastruktur Grunder

Vad är Persistens?

Data som lagras över tid på ett permanent lagringsmedium (non-volatile)

Klarar strömavbrott - data finns kvar när systemet startas om

Data finns kvar även efter att applikationen som skapat det inte finns kvar

Grundläggande för stateful-applikationer, databaser och långtidslagring

Medium för Persistens

HDD - Hard Disk Drives (magnetskiva)

SSD - Solid State Disk (flash-minne, inga rörliga delar)

Optiska Medium - CD, DVD för arkivering

Magnetband - Oftast för backup och långtidslagring

Molnlagring - Amazon S3, Azure Blob Storage

Typer av Persistens

Block Storage: För filsystem. Hårddiskar eller SAN (iSCSI)

File Storage: Hanterar filer och kataloger likt ett filsystem. NAS (SMB, CIFS, NFS)

Object Storage: För objekt, likt filer, men utan hierarki i kataloger. (HTTP/S)

Databaser: Lagrar strukturerad data för enkel åtkomst och manipulation

Persistens i Molnmiljöer

Object Storage: Skalbar lagring för ostrukturerad data (AWS S3, Azure Blob Storage)

Block Storage: Används för databaser och applikationer (AWS EBS, Azure Disk Storage)

Databastjänster: Hanterade tjänster för strukturerad data (Amazon RDS, Azure SQL)

Molnleverantörer hanterar replikering, backuper och hög tillgänglighet automatiskt

Utmaningar med Persistens

Datakonsistens: Säkerställa korrekt och tillförlitlig data i distribuerade system

Skalbarhet: Hantera ökande datavolymer utan prestandaförsämring

Latens: Minimera fördröjningar vid datahämtning

CAP-teoremet beskriver avvägningar mellan Consistency, Availability och Partition tolerance

Fördelar med Persistens

Tillförlitlig lagring: Data förblir intakt och tillgänglig när det behövs

Historisk analys: Möjliggör spårning av trender, mönster och historisk data

Katastrofåterställning: Underlättar backuper och återställning efter fel

Grund för efterlevnad, revision och regulatoriska krav

Sammanfattning

Persistens är en kärnkomponent i modern IT. Förståelse för lagringstyper och teknologier säkerställer effektiva och tillförlitliga system.
