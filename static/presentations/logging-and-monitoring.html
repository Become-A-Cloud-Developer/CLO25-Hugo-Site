Advanced Cloud Development • 2026

You Can't Watch Production Logs

"What broke at 14:32 yesterday?" — there is no terminal scrollback in production

Replicas come and go — their stdout disappears with them

The questions you actually need to answer — about users, deploys, time windows — need a query engine, not a scroll

Three Jumps From Console.WriteLine

Structured — log calls record named fields, not formatted strings

Centralised — every container's stdout streams to one queryable store

Correlated — every log line in one request shares one handle

ILogger<T> — Record, Don't Print

Inject by type — the type parameter becomes the category, your filtering handle

Pick a level + a template + values — the framework records all three so a query engine can pull any of them back

Calls also inherit ambient fields from any enclosing scope (we'll see scopes in Act 4)

Where Stdout Actually Goes

KQL Reads Like A Recipe

Three Filters Carry Every Query

Time — where TimeGenerated > ago(N) bounds the window

App — where ContainerAppName == "..." narrows to one workload in the environment

Match — where Log_s has "..." finds the line you care about

One Request, Many Log Lines

A real request fans out — controller, database, an external API call, the response

Eight log lines from eight categories — a pile, not a story

To tell the story end-to-end, every line in the request needs the same handle

The Scope Inherits Down

A Correlation ID Is Portable

One ID per request, generated at the edge — rides every log line in the request via the log scope

The wider ecosystem — Application Insights, OpenTelemetry — builds on the same idea with richer semantics

One pivot in the query engine — find every line for one CorrelationId — and you have the full request story

Where Logs End

"What's the p95 latency over the last hour?" — needs metrics, sampled at intervals

"Show me the request waterfall across services" — needs traces, with parent/child spans

The next exercise wires those in via Application Insights — same workspace, richer signal
