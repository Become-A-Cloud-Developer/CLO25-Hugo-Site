# Week 8 (v.22) — Wrap-up and Examination

Final week of the ACD course. Walk back through the complete flow — local development → Docker → CI/CD → Azure Container Apps with logs, metrics, secrets, an API surface, blob storage, and health checks — and present the finished application.

## Review

Walk back through the Course Book and revisit any chapter that still feels shaky:

- [Part X — Collaboration and Process](/course-book/10-collaboration-and-process/) — Git, PRs, agile, Jira
- [Part VII — Containers](/course-book/7-containers/) — images, Compose, registries
- [Part V — Identity and Security](/course-book/5-identity-and-security/) — auth, Identity, OIDC, secrets
- [Part VIII — DevOps and Delivery](/course-book/8-devops-and-delivery/) — CI/CD, OIDC, Container Apps
- [Part IX — Operations and Observability](/course-book/9-operations-and-observability/) — logging, App Insights, health checks
- [Part VI — Services and APIs](/course-book/6-services-and-apis/) — REST, DTOs, Swagger
- [Part IV — Data Access](/course-book/4-data-access/) — repositories, transactions, object storage

## Preparation

- Finalize hand-in assignments (Inlämningsuppgift 1 and 2)
- Walk through the reflection questions from each week
- Make sure the deployed application runs end-to-end: login, create record, upload PDF, view logs

## Reflection Questions

- How do all the parts connect — local development → Docker → CI/CD → Azure?
- Which design patterns have we used and why?
- Where would you take this application next: multi-region, blue/green deployment, autoscaling, hardening?
