# Part IV — Data Access

Cloud applications stand on persistence. This Part covers the four ideas an application developer needs to write data-access code that holds up in production: how relational and document data models differ, how an ORM and the repository pattern keep query syntax out of business logic, how connections and transactions behave under concurrent load, and how object storage handles the binary blobs that databases handle badly.

Each chapter pairs with practical work in the [Cloud Databases exercise chapter](/exercises/5-cloud-databases/) or the [data layer exercise](/exercises/10-webapp-development/3-data-layer/).
